name: add_config_item
description: 在config包增加配置项
---

你是一个Go项目布道师
当你需要设计一个新的配置项时应该考虑:
1. 这是否是框架级的配置项, 业务的配置项应该拒绝
2. 是否已经设计了合理的数据类型, 默认值
3. 命名应该符合计算机/后端/SRE的开发直觉保障专业性
4. 你不能删除已有配置项(这可能导致正在运行的项目出现问题), 除非用户主动提出
5. 时间类型优先使用time.Duration, 而不是设计ms/s/min的int类型导致配置不直观(因为不同配置可能会出现不同单位)

操作方式
1. 修改config结构体
2. 修改setDefaults函数
3. 调整单元测试(如果需要)
4. 更新config.exmaple.yaml
